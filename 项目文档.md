# SmartMart WMS 超市仓库管理系统

## 一、项目概述

SmartMart WMS 是一套面向超市场景的网页版仓库管理系统，支持商品入库、出库、库存盘点、供应商管理及数据统计等核心业务，提供三级角色权限控制，适合中小型超市日常仓储运营使用。

## 三、角色权限设计

| 角色 | 权限范围 |
|------|----------|
| 管理员 | 全功能访问，含用户管理、库存调整审核、数据备份 |
| 仓库员 | 入库、出库、库存查询、库存盘点录入 |
| 采购员 | 查看库存预警、管理供应商、查看补货建议 |

权限控制通过 Flask 装饰器 + JWT Token 实现，每个接口根据角色进行拦截。

## 四、功能模块说明

### 4.1 商品入库管理
- 新增入库单，关联供应商、商品、数量、批次号、保质期
- 支持批量导入（Excel 格式）
- 入库后自动更新库存数量
- 生成入库操作日志

### 4.2 商品出库管理
- 根据超市收银需求生成出库单
- 选择商品、填写出库数量、指定领用部门
- 出库前校验库存是否充足，不足则弹窗提醒
- 出库后同步扣减库存

### 4.3 库存实时管理
- 按商品分类 / 批次展示库存明细（名称、规格、库存数、保质期剩余天数）
- 支持模糊搜索
- 设置库存上下限：低于下限自动标红预警，高于上限提示滞销风险

### 4.4 库存盘点管理
- 生成盘点单，支持按区域 / 货架筛选商品
- 手动录入实际库存数量
- 系统自动对比账面库存与实际库存，生成盘盈 / 盘亏报表
- 管理员审核后方可调整库存

### 4.5 供应商管理
- 维护供应商信息：名称、联系方式、供应商品类、合作状态
- 支持新增 / 编辑 / 删除
- 入库单中可快速关联选择供应商

### 4.6 数据统计
- 柱状图：各分类商品库存占比
- 折线图：近 30 天入库 / 出库趋势
- 使用 ECharts 实现前端可视化

### 4.7 数据备份
- 定期自动备份库存、出入库记录
- 支持手动导出 Excel 格式备份文件

### 4.8 操作日志
- 记录所有库存变动操作：操作人、时间、操作类型、修改内容
- 便于问题追溯与审计

## 五、数据库设计

### 5.1 用户表 `users`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 用户ID |
| username | VARCHAR(50) | 用户名 |
| password_hash | VARCHAR(255) | 密码哈希 |
| role | ENUM('admin','warehouse','purchaser') | 角色 |
| created_at | DATETIME | 创建时间 |

### 5.2 商品表 `products`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 商品ID |
| name | VARCHAR(100) | 商品名称 |
| category | VARCHAR(50) | 分类 |
| spec | VARCHAR(100) | 规格 |
| unit | VARCHAR(20) | 单位 |
| stock_min | INT | 库存下限 |
| stock_max | INT | 库存上限 |

### 5.3 库存表 `inventory`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 库存ID |
| product_id | INT FK | 商品ID |
| batch_no | VARCHAR(50) | 批次号 |
| quantity | INT | 当前库存数 |
| expire_date | DATE | 保质期 |
| location | VARCHAR(50) | 货架位置 |

### 5.4 供应商表 `suppliers`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 供应商ID |
| name | VARCHAR(100) | 名称 |
| contact | VARCHAR(50) | 联系人 |
| phone | VARCHAR(20) | 电话 |
| category | VARCHAR(100) | 供应商品类 |
| status | ENUM('active','inactive') | 合作状态 |

### 5.5 入库单表 `inbound_orders`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 入库单ID |
| supplier_id | INT FK | 供应商ID |
| product_id | INT FK | 商品ID |
| batch_no | VARCHAR(50) | 批次号 |
| quantity | INT | 入库数量 |
| expire_date | DATE | 保质期 |
| operator_id | INT FK | 操作人ID |
| created_at | DATETIME | 入库时间 |

### 5.6 出库单表 `outbound_orders`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 出库单ID |
| product_id | INT FK | 商品ID |
| quantity | INT | 出库数量 |
| department | VARCHAR(50) | 领用部门 |
| operator_id | INT FK | 操作人ID |
| created_at | DATETIME | 出库时间 |

### 5.7 盘点单表 `stocktake_orders`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 盘点单ID |
| product_id | INT FK | 商品ID |
| book_qty | INT | 账面库存 |
| actual_qty | INT | 实际库存 |
| diff | INT | 差异（盘盈为正，盘亏为负） |
| status | ENUM('pending','approved') | 审核状态 |
| operator_id | INT FK | 操作人ID |
| created_at | DATETIME | 盘点时间 |

### 5.8 操作日志表 `operation_logs`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 日志ID |
| operator_id | INT FK | 操作人ID |
| action_type | VARCHAR(50) | 操作类型 |
| detail | TEXT | 操作详情 |
| created_at | DATETIME | 操作时间 |

## 七、接口规范（RESTful）

所有接口统一前缀 `/api/v1`，使用 JWT Bearer Token 鉴权。

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | /auth/login | 登录获取 Token |
| 商品 | GET/POST | /products | 查询/新增商品 |
| 商品 | PUT/DELETE | /products/:id | 编辑/删除商品 |
| 库存 | GET | /inventory | 查询库存列表 |
| 入库 | GET/POST | /inbound | 查询/新增入库单 |
| 出库 | GET/POST | /outbound | 查询/新增出库单 |
| 盘点 | GET/POST | /stocktake | 查询/新增盘点单 |
| 盘点 | PUT | /stocktake/:id/approve | 管理员审核盘点 |
| 供应商 | GET/POST | /suppliers | 查询/新增供应商 |
| 供应商 | PUT/DELETE | /suppliers/:id | 编辑/删除供应商 |
| 统计 | GET | /stats/inventory | 库存占比统计 |
| 统计 | GET | /stats/trend | 近30天出入库趋势 |
| 日志 | GET | /logs | 查询操作日志 |
| 备份 | GET | /backup/export | 导出 Excel 备份 |

## 八、环境搭建步骤

### 后端
```bash
cd backend
pip install -r requirements.txt
# 配置 config.py 中的数据库连接信息
flask db init && flask db migrate && flask db upgrade
python app.py
```

### 前端
```bash
cd frontend
npm install
npm run dev
```

### 数据库
```sql
CREATE DATABASE smartmart_wms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
